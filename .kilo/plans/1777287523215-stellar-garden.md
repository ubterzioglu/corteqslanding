# Plan: RAG Q&A Chatbot — Ana Sayfa Entegrasyonu

## Özet

Ana sayfadaki mevcut chat alanını RAG servisine bağla. Frontend doğrudan `rag.corteqs.net`'e vurmasın; bunun yerine nginx reverse proxy üzerinden `corteqs.net/api/chat` → `rag.corteqs.net/api/chat` yönlendirmesi yapılsın.

---

## Mimari

```
corteqs.net frontend
       ↓  fetch("/api/chat", { question })
nginx reverse proxy
       ↓  proxy_pass https://rag.corteqs.net/api/chat
RAG servisi
       ↓  { answer, hasContext }
Frontend'de göster
```

---

## Step 1: nginx.conf — `/api/chat` reverse proxy ekle

**Dosya:** `nginx.conf`

`location / { ... }` bloğundan **önce** yeni bir location block ekle:

```nginx
location = /api/chat {
    if ($request_method != POST) {
        return 405;
    }
    proxy_pass https://rag.corteqs.net/api/chat;
    proxy_ssl_server_name on;
    proxy_set_header Host rag.corteqs.net;
    proxy_set_header Content-Type "application/json";
    proxy_set_header Authorization "Bearer ${RAG_API_SECRET}";
}
```

**Not:** nginx envsubst için `${RAG_API_SECRET}` template variable olarak kullanılır.

---

## Step 2: nginx template mekanizması — Docker güncellemesi

Nginx Docker imajı `/etc/nginx/templates/` dizinindeki `.template` dosyalarını otomatik `envsubst` yapar.

### 2a: nginx.conf → nginx.conf.template

**Dosya:** `nginx.conf` dosyasını `nginx.conf.template` olarak yeniden adlandır.

İçeriği aynı kalır, `${RAG_API_SECRET}` placeholder'ı envsubst ile doldurulur.

### 2b: Dockerfile güncelle

**Dosya:** `Dockerfile`

Değişiklik:
```dockerfile
# Eski:
COPY nginx.conf /etc/nginx/conf.d/default.conf

# Yeni:
COPY nginx.conf.template /etc/nginx/templates/default.conf.template
```

---

## Step 3: .env güncelle

**Dosya:** `.env` ve `.env.example`

Eklenecek:
```env
RAG_API_SECRET=  # Optional: RAG service auth secret
```

Coolify/Vercel ortam değişkenlerine de `RAG_API_SECRET` eklenmeli.

## Step 4: Frontend — RagChat bileşeni oluştur

### 4a: API çağrı fonksiyonu

**Dosya:** `src/lib/ragApi.ts` (yeni)

```ts
type RagResponse = {
  answer: string;
  hasContext: boolean;
};

export async function askRag(question: string): Promise<RagResponse> {
  const res = await fetch("/api/chat", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ question }),
  });

  if (!res.ok) {
    throw new Error("RAG service unavailable");
  }

  return res.json();
}
```

### 4b: RagChat bileşeni

**Dosya:** `src/components/chat/RagChat.tsx` (yeni)

Mevcut chat UI stilinde (ChatWindow, ChatMessage vb. ile uyumlu) bir Q&A bileşeni:
- Text input + gönder butonu
- Loading state (typing indicator)
- Cevap mesajı gösterimi
- `hasContext: false` durumunda "üzgünüm, bu konuda bilgi bulamadım" mesajı
- Yeni soru sorabilme (çoklu soru-cevap dizisi)

UI stili mevcut `ChatWindow` ve `ChatMessage` ile tutarlı olacak (rounded bubbles, bot icon, gradient avatar).

## Step 5: Sayfaya entegre et

**Dosya:** `src/pages/Index.tsx`

Mevcut `FAQSection`'dan sonra veya `ChatBot` (kayıt asistanı) yanında `RagChat` bileşonini ekle.

Kullanıcının belirttiği üzere "sayfada var zaten" — mevcut ChatBot bölümüyle birlikte veya FAQ altında konumlandırılacak.

---

## Step 6: Vite dev proxy (geliştirme ortamı)

**Dosya:** `vite.config.ts`

Geliştirme ortamında `/api/chat` isteğini RAG servisine yönlendirmek için proxy ekle:

```ts
server: {
  proxy: {
    '/api/chat': {
      target: 'https://rag.corteqs.net',
      changeOrigin: true,
      rewrite: (path) => '/api/chat',
    },
  },
}
```
