# Schema.org & GEO Optimization Plan for corteqs.net

## Context

- **Framework:** Vite + React SPA (no SSR/prerender)
- **All schema lives in:** `index.html` (single `<script type="application/ld+json">` block, lines 46–140)
- **Current schemas:** Organization, WebSite, WebPage, FAQPage (5 Qs), SpeakableSpecification
- **Critical bugs found:**
  1. `Organization` has no `@id` — but `WebSite` references `@id: https://corteqs.net/#organization` (broken link)
  2. `WebSite` and `WebPage` also lack `@id` — entity graph is disconnected
  3. Founder "Qualtron Sinclair" typed as `@type: Organization` (should be `Person` or separate entity)
  4. Speakable references `.faq-answer` CSS class — **no such class exists** in any component
  5. Instagram URL mismatch: schema says `instagram.com/corteqs`, footer links to `instagram.com/corteqssocial`

## Step 4 — Add BreadcrumbList Schema

**File:** `index.html` (new entry in `@graph` array)

Add after WebPage:
```json
{
  "@type": "BreadcrumbList",
  "@id": "https://corteqs.net/#breadcrumb",
  "itemListElement": [
    {
      "@type": "ListItem",
      "position": 1,
      "name": "Ana Sayfa",
      "item": "https://corteqs.net/"
    }
  ]
}
```

Since this is a single-page landing site, the breadcrumb is simple (Home only). This satisfies Google's breadcrumb requirements for rich results.
