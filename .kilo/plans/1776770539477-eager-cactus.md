## 2. State Machine (useChatMachine Hook)

### Step Tanımları (`lib/chatConfig.ts`)

```typescript
type ChatStep =
  | "welcome"
  | "category"
  | "fullname"
  | "country"
  | "city"
  | "business"      // opsiyonel
  | "field"
  | "email"
  | "phone"
  | "referral_source"
  | "referral_code"  // opsiyonel
  | "offers_needs"   // opsiyonel
  | "documents"      // opsiyonel
  | "consent"
  | "summary"
  | "completed";
```

### State Shape

```typescript
type ChatState = {
  step: ChatStep;
  messages: ChatMessage[];
  data: ChatCollectedData;
  documentFiles: File[];
  loading: boolean;
  error: string | null;
  submitted: boolean;
};

type ChatCollectedData = {
  category: string | null;
  fullname: string | null;
  country: string | null;
  city: string | null;
  business: string | null;
  field: string | null;
  email: string | null;
  phone: string | null;
  referral_source: string | null;
  referral_detail: string | null;
  referral_code: string | null;
  offers_needs: string | null;
  contest_interest: boolean;
  whatsapp_interest: boolean;
  consent: boolean;
};

type ChatMessage = {
  id: string;
  role: "bot" | "user";
  content: string;
  timestamp: number;
  quickReplies?: QuickReply[];
  isSummary?: boolean;
};

type QuickReply = {
  label: string;
  value: string;
};
```

### Reducer Actions

```typescript
type ChatAction =
  | { type: "SEND_MESSAGE"; payload: string }
  | { type: "SELECT_QUICK_REPLY"; payload: string }
  | { type: "SELECT_CATEGORY"; payload: string }
  | { type: "SELECT_SOURCE"; payload: string }
  | { type: "UPLOAD_FILES"; payload: File[] }
  | { type: "REMOVE_FILE"; payload: number }
  | { type: "TOGGLE_CONSENT"; payload: boolean }
  | { type: "CONFIRM_SUBMIT" }
  | { type: "SUBMIT_SUCCESS" }
  | { type: "SUBMIT_ERROR"; payload: string }
  | { type: "GO_BACK" }
  | { type: "RESET" };
```

### Step Transition Map

```
welcome → category
category → fullname
fullname → country
country → city
city → business (opsiyonel, "yok"/"geç" ile skip)
business → field
field → email
email → phone
phone → referral_source (opsiyonel)
referral_source → referral_code (opsiyonel, "geç" ile skip)
referral_code → offers_needs (opsiyonel, "geç" ile skip)
offers_needs → documents (opsiyonel, "geç" ile skip)
documents → consent
consent → summary
summary → completed (onay üzerine)
```

### Required vs Optional Steps

| Step | Zorunlu | Skip İzin | Validasyon |
|------|---------|-----------|------------|
| category | ✅ | ❌ | `categoryOptions` içinde olmalı |
| fullname | ✅ | ❌ | min 2 kelime, sayı ağırlıklı olamaz |
| country | ✅ | ❌ | min 2 karakter |
| city | ✅ | ❌ | min 2 karakter |
| business | ❌ | ✅ "yok"/"geç" | — |
| field | ✅ | ❌ | min 2 karakter |
| email | ✅ | ❌ | `^[^\s@]+@[^\s@]+\.[^\s@]+$` |
| phone | ✅ | ❌ | E.164: `^\+[1-9]\d{7,14}$` |
| referral_source | ❌ | ✅ | — |
| referral_code | ❌ | ✅ "geç" | max 32 karakter, uppercase |
| offers_needs | ❌ | ✅ "geç" | max 1000 karakter |
| documents | ❌ | ✅ "geç" | dosya validasyonu mevcut utility'den |
| consent | ✅ | ❌ | true olmalı |

## 5. DB Insert — Mevcut `toSubmissionInsert()` ile Uyumluluk

Chatbot verileri `toSubmissionInsert()` fonksiyonuna uygun formatta hazırlanır:

```typescript
// ChatBot submit logic (useChatMachine içinde)
async function submitChatRegistration(state: ChatState) {
  const { data, documentFiles } = state;

  // Dosya upload (mevcut utility)
  const uploadedDocs = await uploadSubmissionDocuments(documentFiles);

  // FormData benzeri values objesi oluştur (toSubmissionInsert ile uyumlu)
  const values: Record<string, FormDataEntryValue> = {
    category: data.category ?? "",
    fullname: data.fullname ?? "",
    country: data.country ?? "",
    city: data.city ?? "",
    business: data.business ?? "",
    field: data.field ?? "",
    email: data.email ?? "",
    phone: data.phone ?? "",
    referral_source: data.referral_source ?? "",
    referral_detail: data.referral_detail ?? "",
    referral_code: data.referral_code ?? "",
    offers_needs: data.offers_needs ?? "",
    document_url: uploadedDocs[0]?.url ?? "",
    document_name: uploadedDocs[0]?.name ?? "",
    documents: uploadedDocs as unknown as FormDataEntryValue,
    contest_interest: data.contest_interest ? "yes" : "",
    whatsapp_interest: data.whatsapp_interest ? "yes" : "",
  };

  const payload = toSubmissionInsert(values, "register");
  const { error } = await supabase.from("submissions").insert(payload);
  if (error) throw error;

  // E-posta bildirimi (mevcut utility)
  const notificationPayload = { ...payload, created_at: new Date().toISOString() };
  await notifySubmission(notificationPayload);
}
```

**Not**: `toSubmissionInsert()` zaten `form_type: "register"`, `status: "new"`, `consent: true` set ediyor. Chatbot'tan gelen veriler bu fonksiyona hiçbir değişiklik yapmadan geçirilir.

## 7. Step-by-Step Implementation

### Step 1 — `lib/chatConfig.ts`
- Step type tanımları
- Her step için bot mesajı, quick replies, validasyon fonksiyonları
- `REQUIRED_STEPS`, `OPTIONAL_STEPS`, `STEP_ORDER` sabitleri
- Validasyon fonksiyonları: `validateName()`, `validateEmail()`, `validatePhone()`
- Komut algılama: `isCommand(input)` → "geri" | "geç" | "iptal" | null

### Step 2 — `hooks/useChatMachine.ts`
- `useReducer` ile state machine
- `sendMessage()` — input'u işle, validasyon yap, next step'e geç
- `selectQuickReply()` — quick reply seçimini işle
- `goBack()` — önceki step'e dön
- `reset()` — session sıfırla
- `submit()` — Supabase insert + e-posta bildirimi
- localStorage persistence (opsiyonel — sayfa yenileme durumunda)

### Step 3 — `components/chat/ChatMessage.tsx`
- Bot/user mesaj balonu
- Quick reply butonları (varsa)
- Markdown-lite destek (bold: **text**)

### Step 4 — `components/chat/ChatInput.tsx`
- Text input + Enter/submit
- Dosya eki butonu (conditional)

### Step 5 — `components/chat/ChatQuickReply.tsx`

### Step 6 — `components/chat/ChatProgressBar.tsx`

### Step 7 — `components/chat/ChatWindow.tsx`

## 8. Dependency / Conflict Analizi

| Risk | Etki | Çözüm |
|------|------|-------|
| Mevcut `RegisterInterestForm` çakışması | Yok | ChatBot ayrı component, form ayrı dialog. İkisi yan yana çalışır |
| `categoryOptions` değişirse | Düşük | ChatBot mevcut `categoryOptions`'ı import eder. Merkezi kaynak |
| `toSubmissionInsert()` imzası değişirse | Düşük | ChatBot bu fonksiyona bağlı. İmza sabit kalır |
| localStorage session çakışması | Yok | Key: `corteqs_chat_session`. Tekil kullanıcı |
| TypeScript strict mode kapalı | Dikkat | Mevcut config `strict: false`. Null check'ler manuel yapılmalı |
| Mobil performans | Düşük | Chat history max 50 mesaj limit koy |
